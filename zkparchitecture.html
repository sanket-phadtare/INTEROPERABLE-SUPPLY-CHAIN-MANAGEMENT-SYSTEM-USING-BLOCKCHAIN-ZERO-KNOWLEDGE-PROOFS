Fig. 1. End-to-end Zero-Knowledge Proof (ZKP) pipeline for privacy-preserving cross-chain attribute verification. The pipeline spans six stages: (1) private input preparation, (2) ZK circuit execution, (3) witness generation, (4) Groth16 proof generation, (5) public signal extraction, and (6) on-chain verification.

1. Inputs to ZKP Circuit
→
🌿 Leaf Hashes
(Poseidon)
→
🌳 Merkle Proof
(Sibling Hashes)
→
🔗 On-chain Merkle Root
(from Polygon)
→
📍 Attribute Index
(Position)
→
🕐 (Optional) IPFS CID / Timestamp
↓
📄 input.json (Private Input)
2. ZK Circuit (Circom)
⟨/⟩ Load Circuit
(.circom)
↓
Recompute Leaf Hash Using Poseidon
↓
Rebuild Merkle Path Up to Root
↓
Compute Root Inside Circuit
↓
Check Equality
(Computed Root ==
On-chain Root)
↓
Enforce Constraints
↓
📄 circuit.circom (ZK Circuit)
3. Witness Generation
📁 Collect Private Inputs
↓
💻 Witness Generator
(snarkjs)
↓
📋 Generate Witness
(.wtns)
↓
Witness Contains:
  • Attribute Values
  • Salts
  • Merkle Path Siblings
  • (Any other private data)
↓
📄 witness.wtns (Witness File)
4. Proof Generation
🛡️ Prover
(snarkjs)
↓
Use Circuit (.r1cs/.wasm)
+ Witness (.wtns)
+ Proving Key (.skey)
↓
📦 Groth16 Proof Generation
↓
✅ Proof Generated
(proof.json)
↓
📄 proof.json (proof.json)
5. Public Signals (Outputs)
📢 Generate Public Signals
(Visible to Verifier)
↓
Public Signals Include:
  • On-chain Merkle Root
  • Attribute Index
  • (Optional) IPFS CID / Timestamp
↓
📄 public.json (Public Signals File)
6. Ready for Verification

(BNB Chain)
↓
🖥️ Send to Verifier Smart Contract
↓
Verifier Uses: Proof + Public Signals
+ Verification Key
↓
✅ Verification Result:
Authentic / Tampered
